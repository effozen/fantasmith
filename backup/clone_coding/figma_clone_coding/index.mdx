## 배경

[따라따라 프로젝트](/challenge/ddara/canvas_and_map_linking/)를 진행할 당시에는 현재에 비해서 많은 지식이 부족했었다.

그러다보니, 구현하기 급급했었고, 원리를 깊게 이해하지 못하고 넘어가는 경우가 많았다.

심지어 다시금 리팩토링을 하려고 해도, 백과 소켓 통신 등에 대해서는 이해가 부족했기에, 리팩토링을 하기가 어려웠다.

최근에 진행하게 된 인턴 생활 역시, 이런 기술적 도메인이 필요했다.

그래서, 어떻게 하면 효과적으로 배워갈 수 있을까 고민을 했었다.

프로젝트를 다시한번 만들까하다가, 실무를 하는 김에 이번에는 체계적으로 <strong>"학습과 성장"</strong>에만 집중하자는 생각이 들어서 피그마 클론 코딩을 진행해보게 되었다.

## 목표

- 피그마의 핵심 기능을 클론코딩하여 실시간 그래픽 편집 기능을 구현한다.
- 소켓 프로그래밍을 활용하여 멀티플레이어 모드 및 서버 동기화를 지원한다.
- Zustand를 기반으로 상태 관리를 설계하고, FSD 구조를 적용하여 유지보수성을 높인다.
- Node.js/Express를 이용해 백엔드 API를 설계하고, Firebase/Supabase 등을 활용해 데이터 저장을 구현한다.
- HTML5 Canvas를 직접 활용하여 선 그리기, 도형 그리기, 이미지 연동 등의 그래픽 기능을 구현한다.
- Excalidraw와 유사하지만 협업 기능이 포함된 버전을 목표로 하며, 외부 그래픽 라이브러리를 사용하지 않고 순수하게 개발한다.

## 학습 목표

- HTML5 Canvas API 및 Web Graphics
- Socket.io를 활용한 실시간 데이터 동기화
- Zustand를 기반으로 한 상태 관리 패턴
- Node.js / Express를 활용한 백엔드 API 설계 및 서버 구축
- Firebase, Supabase 등을 이용한 데이터 저장 및 연동
- Feature-Sliced Design(FSD) 기반의 구조 설계 및 ESLint 규칙 적용
- pnpm-workspace 기반의 모노레포 환경 구성 및 프로젝트 관리
- Vite + TypeScript 환경에서의 최적화된 프론트엔드 개발
- WebSocket과 HTTP API의 차이점 및 효율적인 데이터 통신 방식
- 코멘트 및 히스토리 기능을 구현하며 데이터 변경 이력 관리 방법 학습
- Vercel 또는 Netlify를 이용한 배포 및 CI/CD 경험

## 어떻게 진행할 계획인가?

| 단계                           | 내용                                                               |
| ------------------------------ | ------------------------------------------------------------------ |
| **1. 환경 설정**               | pnpm-workspace 기반 모노레포 설정, Vite + TypeScript 프로젝트 구성 |
| **2. 기본 렌더링 구현**        | HTML5 Canvas API를 활용한 기본 그래픽 렌더링 (선, 도형, 이미지)    |
| **3. 상태 관리 설계**          | Zustand를 이용한 전역 상태 관리 및 FSD 구조 적용                   |
| **4. 사용자 입력 처리**        | 마우스 이벤트 및 키보드 이벤트를 활용한 그래픽 조작 기능 구현      |
| **5. 소켓 프로그래밍 기초**    | Socket.io를 이용한 실시간 동기화 및 이벤트 처리                    |
| **6. 백엔드 구축**             | Node.js + Express 기반 API 서버 및 Firebase/Supabase 데이터 연동   |
| **7. 멀티플레이어 기능**       | 실시간 협업을 위한 WebSocket 기반 동기화 시스템 구현               |
| **8. 코멘트 및 히스토리 기능** | 사용자의 변경 사항을 저장하고 히스토리 및 코멘트 시스템 구축       |
| **9. 최적화 및 성능 개선**     | 렌더링 성능 개선, 배치 업데이트 및 Web Worker 활용                 |
| **10. 배포 및 문서화**         | Vercel 또는 Netlify를 이용한 배포 및 프로젝트 문서화               |

위와 같은 순서로 진행하고자 한다.
