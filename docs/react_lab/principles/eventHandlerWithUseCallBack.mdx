## 🤔 의문의 시작

리액트를 조금씩 깊게 이해할 수록, 아무생각없이 사용하던 것들이 한번씩 툭툭 걸리는 경우가 있다.<br/>

이번이 바로 그런 경우였는데, `이벤트 핸들러` 그리고, `useCallback` 이 바로 그 주제였다.<br/>

의문은 다음과 같았다.<br/><br/>

1. `이벤트 핸들러`의 이름은 어떻게 지어야 하는가?
2. `이벤트 핸들러`를 어디에 배치해야 하는가?
3. `이벤트 핸들러`를 만들 때, `useCallback`을 사용해야 하는가?
4. 이벤트 전달 시에 `()=>{여기에 이벤트 핸들러 넣기}`와 (이벤트핸들러 넣기)의 차이점은 무엇인가?

<br/><br/>

이런 의문들이 생겼다.<br/>

이제 하나씩 알아보도록 하자.<br/><br/>

## 📝 의문 1. `이벤트 핸들러`의 이름은 어떻게 지어야 하는가?

이벤트 핸들러를 구현하다보면, 두 가지 이름 패턴을 보이는 경우가 많았다.<br/>
1. `handle`로 시작하는 이름
2. `on`으로 시작하는 이름

<br/>

이 둘의 차이가 문뜩 궁금해졌다.<br/>

과연 이 둘은 무슨 차이란 말인가? 그래서 검색 등을 통해 찾아보았고 다음과 같은 답을 얻을 수 있었다.<br/>
